# Contributing to Birouter

Thank you for your interest in contributing to Birouter! We welcome contributions from the community to help make Birouter the best AI routing gateway.

## Ways to Contribute

1.  **Reporting Bugs:** If you find a bug, please open an issue on GitHub with a detailed description and steps to reproduce.
2.  **Suggesting Features:** Have an idea for a new feature? Open an issue to discuss it.
3.  **Improving Documentation:** Help us make our documentation better by fixing typos, improving clarity, or adding new sections.
4.  **Submitting Pull Requests:** If you'd like to contribute code, please follow these steps:
    - Fork the repository.
    - Create a new branch for your changes.
    - Make your changes and ensure they follow the project's coding standards.
    - Write tests for your changes if applicable.
    - Submit a pull request with a clear description of your changes.

## Coding Standards

- We use ESLint to maintain code quality. Please ensure your code passes the linting checks.
- Follow the existing code style and patterns in the project.
- Write clear and concise commit messages.

## Developing New Features

If you want to add a new provider or a new translator, please refer to the [Developer Guide](DEVELOPER.md).

## Community

Join our community to discuss the project and get help:
- [GitHub Issues](https://github.com/IQ-Kat/birouter/issues)

Thank you for your contributions!
